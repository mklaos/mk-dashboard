## 1. ບົດສະຫຼຸບຫຍໍ້ (EXECUTIVE SUMMARY)

ບົດສະເໜີນີ້ແມ່ນການນຳສະເໜີທາງອອກທີ່ຄົບຖ້ວນ ເພື່ອເຮັດໃຫ້ການລວບລວມຂໍ້ມູນການຂາຍຈາກທຸກສາຂາຂອງຮ້ານອາຫານ MK ໃນລາວເປັນລະບົບອັດຕະໂນມັດ. ລະບົບນີ້ຈະປ່ຽນການລາຍງານແບບເດີມທີ່ໃຊ້ຄົນເຮັດ ມາເປັນໜ້າຈໍສະແດງຜົນອັດສະລິຍະແບບເຣຍໄທມ໌ (Real-time dashboard) ທີ່ສາມາດເຂົ້າເຖິງໄດ້ຜ່ານອຸປະກອນມືຖື.

## 2. ສິ່ງທ້າທາຍໃນປັດຈຸບັນ (CURRENT CHALLENGES)

### 2.1 ຄວາມຊັກຊ້າຂອງຂໍ້ມູນ (Data Latency)
- ຂໍ້ມູນການຂາຍຖືກເກັບໄວ້ໃນລະບົບຂອງແຕ່ລະສາຂາຈົນກວ່າຈະປິດຮ້ານ.
- ບໍ່ສາມາດເບິ່ງຜົນການດຳເນີນງານໄດ້ຈົນກວ່າຈະຮອດມື້ຖັດໄປ.
- ການປຽບທຽບລະຫວ່າງສາຂາຕ້ອງໃຊ້ການລວບລວມຂໍ້ມູນດ້ວຍມື.
- **ມຸມມອງຜູ້ບໍລິຫານ:** ປັດຈຸບັນ, ທ່ານປະທານສາມາດເຫັນຜົນງານຂອງມື້ວານນີ້ໄດ້ພຽງແຕ່ໃນມື້ນີ້ເທົ່ານັ້ນ.

### 2.2 ຂໍ້ຈຳກັດຂອງການເຮັດວຽກດ້ວຍມື (Manual Bottleneck)
- ທີມງານບັນຊີຕ້ອງໃຊ້ເວລາຫຼາຍຊົ່ວໂມງໃນແຕ່ລະມື້ເພື່ອເກັບກຳ ແລະ ລວບລວມຂໍ້ມູນ.
- ມີຄວາມຜິດພາດຈາກຄົນ (Human errors) ໃນການຄັດລອກຂໍ້ມູນ.
- ການລາຍງານການເງິນມີຄວາມຊັກຊ້າ.

## 4. ແຜນການດຳເນີນງານ (IMPLEMENTATION PLAN)

ໂຄງການຖືກແບ່ງອອກເປັນສາມໄລຍະການສົ່ງມອບທີ່ຊັດເຈນ. ການຊຳລະເງິນແມ່ນຜູກພັນກັບການສຳເລັດຂອງແຕ່ລະຂັ້ນຕອນ.

### ການສົ່ງມອບທີ 1: ຕົວແບບທົດລອງ (Functional Prototype)
- ສຳເລັດການຕັ້ງຄ່າຖານຂໍ້ມູນ ແລະ ລະບົບຄລາວ
- ໂມດູນວິເຄາະໄຟລ໌ XLS (ທັງໝົດ 17 ຮູບແບບ)
- ຕົວແທນທ້ອງຖິ່ນ (Tray App) ພ້ອມຄຸນລັກສະນະ 'Sync Now'
- ໜ້າຈໍຫຼັກຂອງແອັບມືຖື ແລະ ການເຊື່ອມຕໍ່ຂໍ້ມູນຈິງ
- **ເປົ້າໝາຍ:** ທ່ານປະທານສາມາດເຫັນຂໍ້ມູນຈິງຂອງສາຂາເທິງມືຖືໄດ້.

### ການສົ່ງມອບທີ 2: ການປັບປຸງ ແລະ ເພີ່ມປະສິດທິພາບ (Refinement)
- ປັບປຸງຕາມຄຳເຫັນຈາກທ່ານປະທານ ແລະ ທີມບໍລິຫານ
- ແຜນພູມລາຍລະອຽດ, ແຜນຜັງຄວາມຮ້ອນຊ່ວງເວລາຄົນຫຼາຍ, ແລະ ການວິເຄາະແນວໂນ້ມ
- ລະບົບແຈ້ງເຕືອນລາຍການ Void ແລະ ຂໍ້ຜິດພາດ
- **ເປົ້າໝາຍ:** ລະບົບຖືກປັບແຕ່ງໃຫ້ກົງກັບຄວາມຕ້ອງການຂອງຜູ້ບໍລິຫານ MK ຢ່າງແທ້ຈິງ.

### ການສົ່ງມອບທີ 3: ການຕິດຕັ້ງທຸກສາຂາ ແລະ ເລີ່ມໃຊ້ຈິງ (Full Branch Rollout)
- ຕິດຕັ້ງຢູ່ທັງ 3 ສາຂາຂອງ MK
- ເຝິກອົບຮົມພະນັກງານກ່ຽວກັບການ Export Excel ແລະ ຂັ້ນຕອນ 'Sync Now'
- ທົດສອບລະບົບຂັ້ນສຸດທ້າຍ ແລະ ໄລຍະການຊ່ວຍເຫຼືອ 3 ເດືອນ
- **ເປົ້າໝາຍ:** ທັງອົງກອນເລີ່ມໃຊ້ລະບົບ ແລະ ຂັບເຄື່ອນດ້ວຍຂໍ້ມູນ.

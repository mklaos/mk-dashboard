# MK Restaurants - Backend

This folder contains the database schema and API configuration for Supabase.

## Setup Instructions

### 1. Create Supabase Account

1. Go to [supabase.com](https://supabase.com)
2. Sign up for a free account
3. Create a new project named `mk-restaurants`

### 2. Run Database Schema

1. In Supabase dashboard, go to **SQL Editor**
2. Create a new query
3. Copy the entire contents of `db/schema.sql`
4. Click **Run** to execute

### 3. Get API Keys

After creating the project, you'll need:

| Key | Where to Find |
|-----|---------------|
| Project URL | Settings → API → Project URL |
| Anon Key | Settings → API → Project API keys → anon public |
| Service Role Key | Settings → API → Project API keys → service_role |

### 4. Configure Environment

Create `.env` file in backend folder:

```env
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_KEY=your-service-role-key
```

## API Endpoints (Auto-generated by Supabase)

### Query Examples

```sql
-- Today's sales by branch
SELECT * FROM v_branch_performance 
WHERE sale_date = CURRENT_DATE;

-- Peak hours today
SELECT * FROM v_peak_hours 
WHERE sale_date = CURRENT_DATE
ORDER BY total_sales DESC;

-- Recent voids
SELECT * FROM v_void_summary 
ORDER BY sale_date DESC 
LIMIT 20;
```

### REST API Examples

```bash
# Get all branches
curl "https://your-project.supabase.co/rest/v1/branches" \
  -H "apikey: your-anon-key" \
  -H "Authorization: Bearer your-anon-key"

# Get today's sales
curl "https://your-project.supabase.co/rest/v1/daily_sales?sale_date=eq.2026-02-17" \
  -H "apikey: your-anon-key" \
  -H "Authorization: Bearer your-anon-key"

# Insert daily sales
curl -X POST "https://your-project.supabase.co/rest/v1/daily_sales" \
  -H "apikey: your-service-key" \
  -H "Authorization: Bearer your-service-key" \
  -H "Content-Type: application/json" \
  -H "Prefer: return=minimal" \
  -d '{
    "branch_id": "uuid-here",
    "sale_date": "2026-02-17",
    "gross_sales": 35887500,
    "net_sales": 32625273,
    "receipt_count": 54,
    "customer_count": 127
  }'
```

## Security

- Row Level Security (RLS) enabled on all data tables
- Only authenticated users can read/write data
- Service role key required for agent uploads
- Anon key for mobile app read-only access
